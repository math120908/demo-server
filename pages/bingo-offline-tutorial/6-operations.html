Workflow & Operations Guide

How to build, deploy, and operate bingo-offline. Covers build commands, Airflow DAGs, grid configuration, common workflows, and monitoring.

⚙

Build & Release

Build (includes tests)

Full clean build with validation. Tests are run as part of the build command — there is no separate test command.

rexec "mint clean && mint build && mint validate"

Snapshot & Release

Build, validate, generate a snapshot, and publish a release artifact.

rexec "mint build && mint validate && mint snapshot && mint release"

Key Artifacts

bingo-offline-azkaban_2.12 bingo-indexer-azkaban_2.12 Spark 3.1.1

bingo-offline-azkaban_2.12 — Main offline ETL jar. bingo-indexer-azkaban_2.12 — Indexer jar.

Common Operations

  1. Go to Airflow: Search for bingoEtl_holdem__bingo-offline
  2. Trigger DAG with parameters:
    • majorVersion: "1"
    • minorVersion: "2" (production) or "exp" (experimental)
    • msftInboundVersion: "YYYYMMDD" (e.g., "20231116")
  3. Monitor: DAG timeout is 7 days. Uses Iris plan kg-geo-non-critical for failure alerts.
  4. Pipeline copies data from war to holdem, then runs ~35 Spark jobs.
  1. Run publishBetaGeoDatasetsFlow for beta channel
  2. Run publishGeoDatasetsFlow for versioned release
  3. Dali views update: stz_derived_mp.stz_derived_taxonomy_*
  1. Run venicePush DAG — pushes BingGeoEntities store
  2. Run veniceStandardizedBingGeoPush — pushes StandardizedBingGeo store
  3. Run geoPolygonVenicePush — pushes GeoPolygonInfo store
  4. Venice stores: BingGeoEntities, StylizedGeoNames, StandardizedBingGeo, GeoPolygonInfo
  1. Run generateGeoIndex DAG
  2. Indexes are built using Lucene 6.6.6
  3. Uploaded to Artifactory via ArtifactoryUploaderV2
  4. Run compareGeoIndexes to validate against previous version
  1. Edit bingo-offline-azkaban/src/main/resources/custom_data/1_2/market_areas.csv
  2. CSV format: lid,status,remapLid,name,aliases,countryISOCode,bingoIds,parentIds,population,lat,lon,comments,referenceId
  3. Use next available LID in 90000xxx range
  4. Set status=ACTIVE
  5. Build, release, and re-run bingoEtl

Purged entities are entities removed from the Bing data dump. bingo-offline preserves purged entity LIDs for backward compatibility (AppendPurgedEntities).

extractNewlyPurgedIds → enrichNewlyPurgedIds → generateRemapCandidates

Manual review of remap candidates may be needed.

  1. Edit custom_data/1_2/entity_name_hotfix.csv
  2. Format: entityId, locale, name, nameRole
  3. Applied during EntityNameHotfix step in pipeline

Data Versioning

/jobs/msftgeo/versions/{majorVersion}/{majorVersion}_{minorVersion}/
                 |
                 +-- 1/
                      |
                      +-- 1_2/     (production)
                      +-- 1_exp/   (experimental)
                      +-- 1_test/  (user testing)

majorVersion

Currently "1" — changes only for major schema changes.

minorVersion

"2" for production, "exp" for development/experimental.

Version Assignment Rules

Condition minorVersion
USER in [None, "app", "runner", "tester"] 2 (production)
Any other user exp (experimental)

Analysis data path: /jobs/msftgeo/analysis/{major}/{major}_{minor}/
